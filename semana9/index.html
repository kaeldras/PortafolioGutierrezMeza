Semana 9: Desarrollo Backend – Arquitectura y Principios

Descripción de temas aprendidos

En esta semana, profundizamos en el mundo del Desarrollo Backend, comprendiendo su papel fundamental dentro de la arquitectura cliente-servidor que sustenta las aplicaciones web modernas. Aprendimos que el backend es la parte del sistema que se ejecuta en el servidor y se encarga de gestionar la lógica de negocio, la base de datos y la comunicación con el frontend.

Se estudiaron diferentes tipos de arquitecturas web:

  • Arquitectura C/S Clásica – Múltiples Páginas (MPA): Aplicaciones web tradicionales donde cada página es un archivo HTML independiente y la navegación implica recargar la página.
  • Arquitectura C/S Multicapa – Múltiples Páginas: Uso de capas (como el modelo MVC) para separar responsabilidades (datos, lógica, presentación), mejorando el mantenimiento y escalabilidad.
  • Arquitectura SPA (Single Page Application): Aplicaciones de página única que cargan una sola vez el HTML, CSS y JS, y luego actualizan dinámicamente el contenido mediante JavaScript, ofreciendo una experiencia más fluida.
  • Arquitecturas Híbridas (MPA/SPA): Combinan lo mejor de ambos mundos, como en frameworks modernos como Next.js.
  • Arquitectura Hexagonal: Propone separar la lógica central de la aplicación de los detalles externos (bases de datos, interfaces, APIs) mediante puertos y adaptadores, facilitando pruebas y cambios tecnológicos.

También exploramos conceptos clave del entorno backend:

  • Servidores Web: Como Apache, Nginx, IIS y Tomcat, que reciben solicitudes HTTP y sirven contenido.
  • Hosting y Cloud Computing: Diferentes tipos de hosting (compartido, VPS, dedicado, en la nube) y cómo el cloud computing permite escalabilidad bajo demanda.
  • Configuración de servidores: Parámetros como DocumentRoot, Listen, ServerName, DirectoryIndex y seguridad con SSL/TLS.
  • Stacks Backend: Tecnologías como LAMP, MEAN, MERN, y herramientas como Maven y Gradle para gestión de proyectos Java.
  • Maven: Su ciclo de vida (compile, test, package, install, deploy) y el archivo pom.xml como centro de configuración del proyecto.

Ejercicios de laboratorio – Resultados

Como parte del laboratorio, realizamos los siguientes ejercicios prácticos:

  • Instalación de OpenJDK 19 y configuración de las variables de entorno JAVA_HOME y PATH.
  • Instalación de IntelliJ IDEA Community Edition como entorno de desarrollo integrado (IDE) para Java.
  • Instalación del servidor Apache Tomcat 10 desde el instalador de Windows, configurando su conexión con el JDK instalado.
  • Verificación del funcionamiento de Tomcat accediendo a http://localhost:8080 y visualizando la página de bienvenida del servidor.
  • Creación de un proyecto web básico en IntelliJ IDEA y configuración para despliegue en Tomcat.

Estos pasos nos permitieron tener un entorno de desarrollo backend funcional, listo para comenzar a crear aplicaciones web dinámicas con Java y JSP.

Reflexión: ¿Qué aprendí? y ¿Cómo aprendí?

Esta semana amplió significativamente mi visión del desarrollo web. Aprendí que el backend no es solo "lo que está detrás", sino el corazón de una aplicación, responsable de su funcionalidad, seguridad y escalabilidad. Me impresionó la complejidad y el orden que imponen arquitecturas como MVC o Hexagonal, que permiten construir sistemas mantenibles. A través de la instalación práctica de JDK, IntelliJ y Tomcat, entendí mejor cómo se conectan las herramientas para formar un entorno de desarrollo completo. Aprendí haciendo: cada error en las variables de entorno o en la configuración de Tomcat fue una oportunidad para investigar, leer documentación y reforzar mi comprensión. Siento que ahora tengo una base sólida para comenzar a desarrollar aplicaciones del lado del servidor.
