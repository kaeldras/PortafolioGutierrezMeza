Semana 16: Proyecto Final – Sistema Inteligente con Python

Descripción del proyecto

En esta semana, presentamos nuestro proyecto final titulado "Gradient Minimal Portfolio" (GMP), un sistema inteligente integral desarrollado en Python que combina inteligencia artificial, backend, base de datos y frontend para ofrecer una solución completa de análisis de datos con imputación de valores faltantes.

El proyecto fue desarrollado siguiendo las instrucciones detalladas en la evaluación final del curso, que exigía el uso de:

  • Backend: Flask + MySQL + IA (Python)
  • Frontend: HTML, CSS, JavaScript y React
  • Arquitectura: Aplicación web inteligente con integración de modelos de IA

Nuestro sistema está inspirado en el artículo científico estudiado en la Semana 11, sobre el modelo DMDI (Deep Learning-Based Missing Data Imputation), que corrige datos faltantes en sistemas de detección de intrusiones. Adaptamos este enfoque a un contexto más amplio, creando una plataforma que puede aplicarse en salud, finanzas o ciberseguridad.

Objetivos del proyecto

  • Desarrollar una aplicación web funcional con Flask como backend y React como frontend.
  • Integrar un modelo de inteligencia artificial basado en autoencoders y Random Forest para reconstruir y clasificar datos.
  • Utilizar MySQL para almacenar usuarios, resultados y registros del sistema.
  • Implementar una API REST para la comunicación entre frontend y backend.
  • Demostrar la capacidad de integrar múltiples tecnologías en un solo sistema coherente.

Arquitectura del sistema

El sistema sigue una arquitectura modular en capas:

  • Frontend (React): Interfaz de usuario moderna, con componentes reactivos, rutas y estilos.
  • Backend (Flask): Servidor que gestiona las solicitudes, la lógica de negocio y la conexión con la IA.
  • Modelos de IA: Autoencoder (PyTorch) para reconstrucción de datos y Random Forest (scikit-learn) para clasificación.
  • Base de datos (MySQL): Almacena usuarios, sesiones, resultados y análisis de sentimientos.

Componentes técnicos detallados

1. Backend con Flask

El backend, desarrollado en Python con Flask, expone una API REST que permite al frontend interactuar con el sistema.

from flask import Flask, request, jsonify
import joblib
import numpy as np

app = Flask(__name__)

# Cargar modelos entrenados
autoencoder = joblib.load('model/autoencoder.pkl')
classifier = joblib.load('model/random_forest.pkl')

@app.route('/predict', methods=['POST'])
def predict():
    data = request.json['data']
    # Reconstruir datos faltantes
    reconstructed = autoencoder.predict(data)
    # Clasificar
    prediction = classifier.predict(reconstructed)
    return jsonify({'prediction': prediction.tolist()})

if __name__ == '__main__':
    app.run(debug=True)

2. Modelo de Inteligencia Artificial

El sistema utiliza dos modelos clave:

  • Autoencoder: Red neuronal entrenada con PyTorch para reconstruir datos faltantes. Se entrena con datos normales y aprende a representar patrones.
  • Random Forest: Clasificador que predice categorías a partir de los datos reconstruidos.

El entrenamiento se realizó en tres fases:

  1. Carga y normalización de datos con StandardScaler.
  2. Entrenamiento del autoencoder durante 20 épocas.
  3. Entrenamiento del Random Forest y guardado de ambos modelos.

3. Base de Datos (MySQL)

Se creó una base de datos gmp_db con tablas para usuarios y resultados:

CREATE DATABASE gmp_db;
USE gmp_db;

CREATE TABLE usuarios (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nombre VARCHAR(100),
    email VARCHAR(100) UNIQUE,
    password VARCHAR(255)
);

CREATE TABLE resultados (
    id INT AUTO_INCREMENT PRIMARY KEY,
    usuario_id INT,
    datos_originales TEXT,
    datos_reconstruidos TEXT,
    prediccion VARCHAR(50),
    fecha TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (usuario_id) REFERENCES usuarios(id)
);

5. Análisis de Sentimientos (TextBlob)

Como funcionalidad adicional, implementamos un análisis de sentimientos usando TextBlob:

from textblob import TextBlob

def analizar_sentimiento(texto):
    blob = TextBlob(texto)
    polaridad = blob.sentiment.polarity
    if polaridad > 0:
        return "Positivo"
    elif polaridad < 0:
        return "Negativo"
    else:
        return "Neutro"

print(analizar_sentimiento("Me encanta este sistema inteligente"))
