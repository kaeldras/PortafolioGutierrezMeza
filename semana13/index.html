Semana 13: Desarrollo Backend – PHP con Laravel

Descripción de temas aprendidos

En esta semana, profundizamos en el desarrollo backend utilizando PHP junto con uno de sus frameworks más populares: Laravel. Aprendimos que PHP (PHP: Hypertext Preprocessor) es un lenguaje de programación del lado del servidor, ampliamente utilizado para crear aplicaciones web dinámicas.

Estudiamos cómo funciona PHP: cuando un usuario solicita una página con código PHP, el servidor lo procesa mediante un intérprete, que genera como resultado código HTML puro, el cual es enviado al navegador. Esto permite crear contenido dinámico como formularios, validaciones, acceso a bases de datos y más.

Luego, ingresamos al mundo de Laravel, un potente framework de PHP que sigue el patrón MVC (Modelo-Vista-Controlador) y que permite desarrollar aplicaciones web de forma más rápida, organizada y segura. Laravel ofrece herramientas modernas que simplifican tareas comunes del desarrollo web.

Los conceptos clave que aprendimos fueron:

  • Composer: El gestor de dependencias de PHP, similar a Maven en Java. Permite instalar Laravel y sus librerías de forma sencilla.
  • Artisan: La herramienta de línea de comandos de Laravel, que automatiza tareas como crear controladores, modelos, migraciones, etc.
  • Rutas (Routes): Definen las URLs de la aplicación y las asocian a un controlador o acción específica.
  • Controladores (Controllers): Agrupan la lógica de negocio y gestionan las solicitudes del usuario.
  • Vistas con Blade: Motor de plantillas de Laravel que permite crear interfaces dinámicas mezclando HTML y PHP de forma limpia.
  • Eloquent ORM: Sistema de mapeo objeto-relacional que permite interactuar con la base de datos usando sintaxis PHP en lugar de SQL directo.
  • Migraciones y Seeds: Permiten gestionar la estructura de la base de datos de forma programática y llenarla con datos de prueba.
  • Middleware: Filtros que se ejecutan antes o después de una solicitud, usados para autenticación, autorización o validación.

Ejercicios de laboratorio – Resultados

Realizamos los siguientes ejercicios prácticos:

  • Ejercicio 1: Crear un proyecto Laravel desde cero.

    Usamos el instalador global de Laravel para crear un nuevo proyecto:

    laravel new estudiantes-app

    Luego, accedimos al directorio y ejecutamos el servidor de desarrollo:

    cd estudiantes-app
php artisan serve

    El proyecto se inició correctamente y fue accesible desde http://localhost:8000.

  • Ejercicio 2: Configurar conexión a MySQL y crear un modelo con migración.

    Editamos el archivo .env para configurar la base de datos:

    DB_DATABASE=academico
DB_USERNAME=root
DB_PASSWORD=tu_contraseña

    Luego, creamos el modelo Estudiante con su migración:

    php artisan make:model Estudiante -m

    Editamos la migración para definir los campos y ejecutamos:

    php artisan migrate

    La tabla estudiantes se creó correctamente en la base de datos.

  • Ejercicio 3: Crear un formulario de registro con Laravel, Blade y Tailwind CSS.

    Configuramos Tailwind CSS para dar estilos modernos a la interfaz:

    npm install -D tailwindcss@3 postcss autoprefixer
npx tailwindcss init -p

    Definimos rutas en routes/web.php, creamos un controlador y una vista con Blade. El formulario permitía registrar estudiantes y guardaba los datos en la base de datos mediante Eloquent.

    El diseño fue limpio y responsivo gracias a Tailwind, y el uso de @csrf garantizó la seguridad del formulario.

Estructura de Directorios de Laravel

Aprendimos que Laravel tiene una estructura de carpetas bien definida que facilita la organización del código:

  • app/: Contiene los modelos, controladores, middleware y providers.
  • routes/: Define las rutas web y API.
  • resources/views/: Almacena las vistas Blade.
  • database/migrations/: Gestiona la estructura de la base de datos.
  • public/: Punto de entrada de la aplicación (index.php).
  • config/: Archivos de configuración (base de datos, sesiones, etc.).
  • storage/: Almacena logs, caché y archivos subidos.
  • vendor/: Dependencias instaladas por Composer.
