Semana 14: Desarrollo Backend – Python con Flask

Descripción de temas aprendidos

En esta semana, nos adentramos en el desarrollo backend utilizando Python junto con uno de sus frameworks más populares: Flask. Aprendimos que Python es un lenguaje de programación interpretado, de tipado dinámico y alto nivel, ampliamente utilizado en áreas como desarrollo web, ciencia de datos, inteligencia artificial y automatización.

Estudiamos las principales características de Python:

  • Sintaxis clara y legible, cercana al lenguaje humano.
  • Indentación con espacios (4 por nivel) en lugar de llaves.
  • Tipado dinámico: No se declara el tipo de variable.
  • Multiparadigma: Soporta programación orientada a objetos, funcional e imperativa.
  • Manejo de archivos, estructuras de control y funciones de forma sencilla.

Luego, ingresamos al mundo de Flask, un microframework web escrito en Python, diseñado para ser ligero, flexible y fácil de usar. Flask no impone una estructura rígida, lo que lo hace ideal para proyectos pequeños o de aprendizaje, pero también escalable con buenas prácticas.

Los conceptos clave que aprendimos fueron:

  • Flask: El núcleo del framework, usado para crear aplicaciones web.
  • Rutas (@app.route): Decoradores que definen las URLs que activan funciones.
  • Request: Objeto que contiene los datos enviados por el cliente (formularios, parámetros, etc.).
  • Response: Datos que la app envía de vuelta al cliente.
  • render_template: Función que renderiza archivos HTML usando Jinja2.
  • Jinja2: Motor de plantillas que permite incrustar lógica en archivos HTML (condicionales, bucles, variables).
  • Blueprints: Para dividir la app en módulos reutilizables.
  • Debug Mode: Modo de desarrollo que recarga automáticamente y muestra errores detallados.
  • Extensiones: Como Flask-SQLAlchemy, Flask-WTF, Flask-Login, que amplían la funcionalidad del framework.

4. Primera aplicación Flask

Instalamos Flask y creamos una app básica:

pip install flask

Archivo app.py:

from flask import Flask

app = Flask(__name__)

@app.route('/')
def hola():
    return '¡Hola Mundo desde Flask!'

if __name__ == '__main__':
    app.run(debug=True)

Al ejecutar python app.py, la app se inicia en http://127.0.0.1:5000/.

Ejercicios de laboratorio – Resultados

Realizamos los siguientes ejercicios prácticos:

  • Ejercicio 1: Crear una clase Persona y Estudiante en Python.

    Implementamos POO creando una clase base Persona y una clase Estudiante que hereda de ella.

    class Persona:
    def __init__(self, nombre, edad):
        self.nombre = nombre
        self.edad = edad

    def presentarse(self):
        return f"Hola, soy {self.nombre} y tengo {self.edad} años."

class Estudiante(Persona):
    def __init__(self, nombre, edad, carrera):
        super().__init__(nombre, edad)
        self.carrera = carrera

    def estudiar(self):
        return f"{self.nombre} está estudiando {self.carrera}."

# Prueba
alumno = Estudiante("Laura", 20, "Ingeniería de Sistemas")
print(alumno.presentarse())
print(alumno.estudiar())
  • Ejercicio 2: Crear una aplicación web con Flask que registre estudiantes en una base de datos MySQL.

    Configuramos una base de datos escuela y una tabla estudiantes.

    CREATE DATABASE escuela;
USE escuela;

CREATE TABLE estudiantes(
    IdEstudiante INT PRIMARY KEY AUTO_INCREMENT,
    nomEstudiante VARCHAR(100),
    dirEstudiante VARCHAR(150),
    ciuEstudiante VARCHAR(100)
);

    Archivo conexion.py para conectar a MySQL:

    import pymysql

def obtener_conexion():
    return pymysql.connect(
        host='localhost',
        user='root',
        password='tu_contraseña',
        db='escuela',
        cursorclass=pymysql.cursors.DictCursor
    )

    Archivo app.py con rutas para formulario y registro:

    from flask import Flask, render_template, request, redirect
from conexion import obtener_conexion

app = Flask(__name__)

@app.route('/')
def formulario():
    return render_template('formulario.html')

@app.route('/registrar', methods=['POST'])
def registrar():
    nombre = request.form['nombre']
    direccion = request.form['direccion']
    ciudad = request.form['ciudad']
    
    conexion = obtener_conexion()
    with conexion.cursor() as cursor:
        cursor.execute(
            "INSERT INTO estudiantes (nomEstudiante, dirEstudiante, ciuEstudiante) VALUES (%s, %s, %s)",
            (nombre, direccion, ciudad)
        )
    conexion.commit()
    conexion.close()
    
    return redirect('/')

if __name__ == '__main__':
    app.run(debug=True)

    Y un formulario HTML en templates/formulario.html:

    La aplicación funcionó correctamente, permitiendo registrar estudiantes en la base de datos desde el navegador.
