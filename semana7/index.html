Semana 7: Uso de Hooks en React

Descripción de temas aprendidos: Los Hooks en React

En esta semana crucial, nos sumergimos de lleno en el concepto y la aplicación de los **Hooks en React**, una característica introducida en la versión 16.8 que revolucionó la forma de escribir componentes funcionales. Los Hooks permiten a estos componentes "engancharse" a las características de estado y ciclo de vida de React, que antes eran exclusivas de los componentes de clase, simplificando enormemente el código y la lógica de las aplicaciones.

¿Qué es un Hook?

Un Hook es una función de JavaScript que permite a los componentes funcionales acceder a las funcionalidades de estado y ciclo de vida de React. Esto significa que podemos añadir estado y manejar efectos secundarios en nuestros componentes sin necesidad de convertirlos en clases, haciendo el código más conciso y fácil de entender.

Tipos de Hooks Fundamentales

Los Hooks más utilizados y esenciales que abordamos fueron:

  • **`useState`**: Este Hook permite añadir estado a componentes funcionales. Retorna un par de valores: el estado actual y una función para actualizarlo. Es fundamental para manejar datos que cambian en el tiempo dentro de un componente, como un contador, el contenido de un input, o la visibilidad de un elemento.

    Ejemplo de uso de `useState`:

    import React, { useState } from 'react';

function Contador() {
  const [cuenta, setCuenta] = useState(0); // Inicializa 'cuenta' en 0

  return (
    <div>
      <p>Has hecho clic {cuenta} veces</p>
      <button onClick={() => setCuenta(cuenta + 1)}>
        Haz clic aquí
      </button>
    </div>
  );
}
  • **`useEffect`**: Se utiliza para manejar efectos secundarios (side effects) en componentes funcionales. Los efectos secundarios son operaciones que ocurren fuera del flujo de renderizado normal de React, como la manipulación directa del DOM, suscripciones a eventos, temporizadores, o la carga de datos desde una API. Se ejecuta después de cada renderizado del componente por defecto, pero se puede controlar su ejecución con un array de dependencias.

    Ejemplo de uso de `useEffect` (actualizando el título del documento):

    import React, { useState, useEffect } from 'react';

function TituloDocumento() {
  const [titulo, setTitulo] = useState('Mi App React');

  useEffect(() => {
    document.title = titulo; // Actualiza el título del documento
    // Opcional: función de limpieza
    return () => {
      // Por ejemplo, restaurar el título original si el componente se desmonta
      // document.title = 'Portafolio Principal';
    };
  }, [titulo]); // Se ejecuta cada vez que 'titulo' cambia

  return (
    <div>
      <input
        type="text"
        value={titulo}
        onChange={(e) => setTitulo(e.target.value)}
      />
      <p>El título actual es: {titulo}</p>
    </div>
  );
}
  • **`useContext`**: Permite acceder al valor de un contexto de React. El Context API es una forma de compartir datos que pueden considerarse "globales" para un árbol de componentes de React, sin tener que pasar props manualmente a través de cada nivel. Es ideal para temas, preferencias de idioma o estado de autenticación.
  • **`useReducer`**: Es una alternativa a `useState` para la gestión de estado más compleja. Es especialmente útil cuando el estado tiene múltiples subvalores o cuando el siguiente estado depende del anterior. Es muy similar a la forma en que Redux gestiona el estado.
  • **`useRef`**: Retorna un objeto ref mutable cuya propiedad `.current` se inicializa con el argumento pasado. Se utiliza comúnmente para acceder a elementos del DOM directamente (sin React), o para almacenar un valor mutable que no cause un re-renderizado cuando cambia (a diferencia del estado).

Hooks de React Router Dom 6

Para la navegación en aplicaciones React que utilizan la librería `React Router Dom` en su versión 6, se introdujeron hooks específicos que simplifican enormemente la gestión de rutas:

  • **`useNavigate`**: Este hook proporciona una función para navegar programáticamente entre diferentes rutas de la aplicación. Es el reemplazo moderno de `useHistory` y es muy versátil.

    Ejemplo:

    import { useNavigate } from 'react-router-dom';

function HomeButton() {
  const navigate = useNavigate();

  const handleClick = () => {
    navigate('/dashboard'); // Navega a la ruta /dashboard
    // navigate(-1); // Para ir a la página anterior en el historial
  };

  return (
    <button onClick={handleClick}>Ir al Dashboard</button>
  );
}
  • **`useParams`**: Permite acceder a los parámetros dinámicos que se encuentran en la URL de la ruta actual. Es muy útil para extraer IDs u otros identificadores de la URL.

    Ejemplo (para una ruta como `/productos/:id`):

    import { useParams } from 'react-router-dom';

function DetallesProducto() {
  const { id } = useParams(); // Si la ruta es /productos/123, id será "123"

  return (
    <div>
      <h2>Detalles del Producto: {id}</h2>
      {/* Aquí puedes cargar los detalles del producto con ese ID */}
    </div>
  );
}

Conceptos Relacionados: Lazy Loading y Suspense

Aunque no son Hooks en sí mismos, "Lazy Loading" y "Suspense" son características de React que a menudo se utilizan en conjunto con los Hooks para optimizar el rendimiento de las aplicaciones:

  • **Lazy Loading (Carga Perezosa)**: Es una técnica para cargar componentes o módulos de forma asíncrona, solo cuando son necesarios. Esto ayuda a reducir el tamaño del paquete inicial de la aplicación, mejorando significativamente el tiempo de carga. Se implementa con `React.lazy()`.
  • **Suspense**: Es un componente de React que permite "esperar" a que el código de un componente cargado perezosamente se descargue y se muestre un indicador de carga (fallback) mientras tanto. Esencial para proporcionar una experiencia de usuario fluida durante la carga asíncrona.

Gestión de Estado en React

La gestión del estado es un pilar en React, y los Hooks han proporcionado herramientas poderosas para abordarla. Se discutieron diferentes enfoques:

  • **Estado Local (`useState`, `useReducer`)**: Para datos que solo afectan a un componente específico.
  • **Context API (`useContext`)**: Para compartir estado de forma global sin prop drilling.
  • **Librerías de Gestión de Estado (ej. Redux, Zustand)**: Para aplicaciones complejas que requieren una gestión de estado más centralizada y predecible.

El estudio de los Hooks ha sido un paso fundamental para escribir código React más moderno, eficiente y fácil de mantener.
