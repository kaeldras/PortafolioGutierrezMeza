Semana 8: Evaluación Parcial – Aplicaciones con React y Vite

Descripción de la actividad

En esta semana, se llevó a cabo la evaluación parcial del curso de Desarrollo de Aplicaciones Web. Esta evaluación tuvo como objetivo medir nuestras habilidades en el desarrollo de aplicaciones web modernas utilizando tecnologías clave como React y Vite, en un entorno de trabajo colaborativo.

La evaluación se realizó bajo las siguientes condiciones:

  • Duración: 60 minutos.
  • Modalidad: Programación por pares (trabajo en pareja).
  • Tecnologías requeridas: React, Vite, JavaScript, componentes funcionales.
  • Entrega: Aplicación funcional con estructura de componentes y manejo de datos.

Enunciado de la evaluación

Según el documento oficial de la evaluación, las instrucciones fueron las siguientes:

1. Desarrollar una aplicación en React y Vite con 3 componentes o más (ComponentGeneral, ListaEstudiantes y Estudiante).

2. En ComponentGeneral, leer los datos de un archivo JSON que contiene información de estudiantes.

3. Mostrar los datos en una lista utilizando el componente ListaEstudiantes.

4. Cada estudiante debe ser representado por el componente Estudiante, que muestre su información de forma clara y estructurada.

Objetivos de la evaluación

Esta evaluación buscaba verificar las siguientes competencias:

  • Uso de React: Creación de componentes funcionales y manejo de props.
  • Manejo de estado y datos: Carga de un archivo JSON y renderizado dinámico de información.
  • Estructura de componentes: División lógica de la interfaz en componentes reutilizables.
  • Trabajo colaborativo: Programación por pares, toma de decisiones en equipo y resolución de problemas en tiempo real.
  • Uso de herramientas modernas: Configuración y ejecución de un proyecto con Vite + React.

Alcance técnico

La aplicación debía cumplir con una estructura mínima como la siguiente:

src/
├── components/
│   ├── ComponentGeneral.jsx
│   ├── ListaEstudiantes.jsx
│   └── Estudiante.jsx
├── data/
│   └── estudiantes.json
└── App.jsx

Donde:

  • ComponentGeneral.jsx: Se encarga de cargar los datos del archivo estudiantes.json y pasarlos como props al componente lista.
  • ListaEstudiantes.jsx: Recibe la lista de estudiantes y la mapea para renderizar múltiples instancias del componente Estudiante.
  • Estudiante.jsx: Muestra la información de un estudiante (nombre, código, carrera, etc.) en una tarjeta o fila.
  • estudiantes.json: Archivo con datos en formato JSON, como: 
    [
  {
    "id": 1,
    "nombre": "Ginao Jhardel",
    "codigo": "2013200634H",
    "carrera": "Ingeniería de Sistemas"
  },
  {
    "id": 2,
    "nombre": "Lucía Mendoza",
    "codigo": "2014200756K",
    "carrera": "Ingeniería Industrial"
  }
]

Reflexión: ¿Qué aprendí? y ¿Cómo aprendí?

Esta evaluación fue un momento clave del curso, ya que puso a prueba nuestras habilidades técnicas bajo presión y en un entorno colaborativo. Aprendí que trabajar en pareja requiere comunicación clara, división de tareas y constante retroalimentación. Aunque no guardé una copia del código final, el proceso de desarrollo me permitió reforzar conceptos clave como la creación de componentes en React, el manejo de datos externos y la importancia de una buena estructura de proyecto.

Aprendí haciendo: tuvimos que configurar Vite, crear componentes, cargar datos JSON y asegurarnos de que todo funcionara en tiempo real. Cada error en el manejo de props o en la ruta del archivo JSON fue una oportunidad para investigar y corregir rápidamente. Sentí una gran satisfacción al ver cómo la lista de estudiantes se mostraba correctamente en pantalla. Esta experiencia me dio confianza para seguir desarrollando aplicaciones web modernas con React.
